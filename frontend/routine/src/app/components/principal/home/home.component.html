Project Structure

This project uses 2 springboot backend api, 'def' api is connected to simple definition tables in the database (such as sport definiton table).

'mainapp' is connected to iteration type tables which performs dynamic operations such as inserting, deleting, etc.

'def' & 'mainapp' mini-services are independently of each other. Both uses a self-made library.

'sharedLibrary' is a jar project acts as library. It only has 3 classes:

  • Constants -> for decided fixed constants
  • DateConverter -> for fixed date conversion
  • EntityToDaoMapper -> converts given entity to dao pojo classes

    • This project uses service-response instead of sending the data directly.

    In service-responses, I have avoided putting entity objects directly inside data attribute, instead I have turned them into 'dao's of specific type
